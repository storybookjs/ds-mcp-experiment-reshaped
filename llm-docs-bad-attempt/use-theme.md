# useTheme Hook

## Overview
The useTheme hook provides access to and manipulation of theme configuration, including design tokens, color modes, and theming utilities throughout the component tree.

## Key Features
- Access to current theme configuration
- Color mode management
- Design token retrieval
- Theme switching capabilities
- Responsive value resolution

## Hook Interface
```typescript
type ThemeContextValue = {
  theme: Theme;
  colorMode: 'light' | 'dark' | 'auto';
  resolvedColorMode: 'light' | 'dark';
  setColorMode: (mode: 'light' | 'dark' | 'auto') => void;
  getToken: (path: string) => any;
  direction: 'ltr' | 'rtl';
};

function useTheme(): ThemeContextValue;
```

Properties\n\n### Color Mode Management\n- **colorMode**: Currently set color mode preference\n- **resolvedColorMode**: Actual color mode being used\n- **setColorMode**: Function to change color mode\n\n### Theme Access\n- **theme**: Complete theme object with all design tokens\n- **getToken**: Function to retrieve specific design token values\n- **direction**: Text direction (ltr/rtl)\n\n## Design Token Categories\n- **color**: All color tokens (primary, background, text, etc.)\n- **spacing**: Spacing scale values\n- **fontSize**: Typography size tokens\n- **radius**: Border radius values\n- **shadow**: Box shadow definitions\n- **breakpoint**: Responsive breakpoint values\n\n## Best Practices\n- Use `getToken` for accessing design tokens\n- Prefer `resolvedColorMode` over `colorMode` for styling\n- Memoize expensive token calculations\n- Use custom hooks for commonly accessed theme values\n- Respect user's color mode preference\n\n## Related Hooks\n- **useRTL**: Text direction detection\n- **useResponsiveClientValue**: Responsive value resolution"
